## Issue #35 — Login page

**Files:**
- `src/qbr_web/auth.py` NEW — session + bcrypt helpers
- `src/qbr_web/app.py` — SessionMiddleware, auth middleware, `/login`, `/logout`
- `src/qbr_web/templates/login.html` NEW
- `src/qbr/cli.py` — `hash-password` command
- `pyproject.toml` — add `bcrypt`, `itsdangerous`
- `.env.example` — `QBR_AUTH_USER`, `QBR_AUTH_PASSWORD_HASH`, `QBR_SESSION_SECRET`
- `README.md` — auth setup section

**Auth flow:**
1. Incoming request → middleware checks `request.session.get("user")`
2. Missing + path not in allowlist → redirect to `/login?next=<original>`
3. Allowlist: `/login`, `/logout`, `/healthz`, `/static/*`
4. POST /login → check bcrypt, set session, redirect to `next` or `/`
5. POST /logout → clear session, redirect to `/login`

**Env var defaults (dev mode):**
- If no auth env vars set → auth disabled (for tests + local dev)
- Production: set all three env vars

**Rate limit:** In-memory dict `{ip: [(timestamp, ...)]}`, max 5 attempts/15 min.

**Tests:**
- Unauthenticated request to `/` → redirect to `/login`
- Valid login → session set, redirect works
- Invalid login → error message shown
- Logout clears session
- Auth disabled mode → all requests work without session
